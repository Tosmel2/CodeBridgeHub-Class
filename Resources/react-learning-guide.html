What is React?

React is a JavaScript library for building user interfaces — created by Facebook in 2013. It lets you build complex UIs from small, isolated pieces of code called components.

⚡ Why React?

React solves the core problem of keeping the UI in sync with data — efficiently and predictably.

  • Component-Based Architecture
  • Virtual DOM for fast updates
  • Declarative UI — describe what you want
  • Huge ecosystem and community
  • Used by Meta, Netflix, Airbnb, Twitter
  • One-way data flow

🌐 React vs Traditional JS

Traditional JSReact
Manually update DOMReact updates DOM automatically
Code gets messy fastOrganized into components
Hard to reuse codeComponents are reusable
Imperative styleDeclarative style
💡 Core Concept React is NOT a framework — it's a library focused only on the View layer (UI). You combine it with other tools for routing, state management, etc.

🏛️ The Virtual DOM

React keeps a lightweight copy of the real DOM in memory — the Virtual DOM. When state changes:


1

Re-render Virtual DOM

React re-renders the component tree in memory (very fast — no painting)

2

Diffing Algorithm

React compares the new Virtual DOM with the previous snapshot to find what changed

3

Patch the Real DOM

Only the changed parts get updated in the actual browser DOM — minimal and efficient

📜 JSX Rules

✅ Rules to Follow

  • Return ONE root element (or Fragment)
  • Use className not class
  • Use htmlFor not for
  • Self-close empty tags: <img />
  • Use camelCase for attributes
  • Inline styles use objects: {{ color: 'red' }}

Components are the building blocks of React apps. They are independent, reusable pieces of UI — like LEGO bricks you snap together to build anything.

🔑 Key Rule Component names MUST start with a capital letter. MyComponent ✅   mycomponent ❌

⚙️ Function Components (Modern)

Props — Passing Data

Props (short for "properties") are how you pass data from a parent component to a child component. They are read-only — a child can never modify its own props.

⚠️ Golden Rule Props flow DOWN — parent to child. Never modify props inside the child. They are immutable (read-only).
UserCard.jsx JSX 
// Child component receives props as an argument
function UserCard(props) {
  return (
    <div className="card">
      <h2>{props.name}</h2>
      <p>Age: {props.age}</p>
      <p>Role: {props.role}</p>
    </div>
  )
}

// ✨ Better: Destructure props directly
function UserCard({ name, age, role = 'User' }) {
  // role = 'User' is the default value
  return (
    <div className="card">
      <h2>{name}</h2>
      <p>Age: {age}</p>
      <p>Role: {role}</p>
    </div>
  )
}

// Parent passes data like HTML attributes
function App() {
  return (
    <div>
      <UserCard name="Alice" age={25} role="Admin" />
      <UserCard name="Bob" age={30} />  // role defaults to 'User'
    </div>
  )
}

function Counter() {
  // useState returns [currentValue, setterFunction]

  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count + 1)}>+1</button>

function ProfileForm() {
  // State can be an object
  const [user, setUser] = useState({
    name: '',
    email: '',
    age: 0
  })

function TodoList() {
  // State as an array
  const [todos, setTodos] = useState(['Learn React', 'Build projects'])

  const addTodo = () => {
    // NEVER mutate state directly — always create new array
    setTodos([...todos, 'New todo'])
  }

  const removeTodo = (index) => {
    setTodos(todos.filter((_, i) => i !== index))
  }

  return (
    <ul>
      {todos.map((todo, i) => (
        <li key={i}>
          {todo}
          <button onClick={() => removeTodo(i)}>✕</button>
        </li>
      ))}
    </ul>
  )
}

React Hooks

Hooks let you "hook into" React features from function components. Introduced in React 16.8, they replaced class components entirely.

⚠️ Rules of Hooks 1. Only call hooks at the top level — not inside loops, conditions, or nested functions.   2. Only call hooks from React function components or custom hooks.
useState

Add reactive state to a component. Returns current value + setter function.

useEffect

Run side effects (data fetching, subscriptions, timers) after render.

useContext

Read from a React Context without prop drilling.

useRef

Access DOM elements directly or store mutable values that don't trigger re-renders.

useReducer

Manage complex state logic with a Redux-style reducer pattern.

useMemo

Memoize expensive calculations so they only recompute when dependencies change.

useCallback

Memoize functions to prevent re-creation on every render.

useId

Generate unique, stable IDs for accessibility attributes.

🔁 useEffect Deep Dive

Runs after every render by default. Use the dependency array to control when it runs.

useEffectExamples.jsx JSX 
import { useState, useEffect } from 'react'

function DataFetcher({ userId }) {
  const [user, setUser] = useState(null)
  const [loading, setLoading] = useState(true)

  // ✅ Runs ONCE after first render (empty array)
  useEffect(() => {
    document.title = 'React App'
  }, [])

  // ✅ Runs when userId changes
  useEffect(() => {
    setLoading(true)

    fetch(`https://api.example.com/users/${userId}`)
      .then(res => res.json())
      .then(data => {
        setUser(data)
        setLoading(false)
      })

    // Cleanup function (runs before next effect or unmount)
    return () => {
      console.log('Cleanup! userId changed or component unmounted')
    }
  }, [userId])   // ← dependency array: re-run when userId changes

  // ✅ Runs after EVERY render (no dependency array)
  useEffect(() => {
    console.log('Component re-rendered!')
  })

  if (loading) return <p>Loading...</p>
  return <div>{user?.name}</div>
}
useRef.jsx JSX

function FocusInput() {
  // useRef creates a mutable ref object { current: initialValue }

useReducer.jsx JSX 
import { useReducer } from 'react'

// Define what actions can happen and how state changes
function counterReducer(state, action) {
  switch (action.type) {
    case 'INCREMENT': return { ...state, count: state.count + 1 }
    case 'DECREMENT': return { ...state, count: state.count - 1 }
    case 'RESET':     return { count: 0 }
    case 'SET':       return { count: action.payload }
    default: return state
  }
}

function Counter() {
  const [state, dispatch] = useReducer(counterReducer, { count: 0 })

  return (
    <div>
      <p>Count: {state.count}</p>
      <button onClick={() => dispatch({ type: 'INCREMENT' })}>+</button>
      <button onClick={() => dispatch({ type: 'DECREMENT' })}>-</button>
      <button onClick={() => dispatch({ type: 'SET', payload: 100 })}>Set 100</button>
    </div>
  )
}

Component Lifecycle

Every component goes through a lifecycle: Mount → Update → Unmount. With hooks, you control lifecycle behavior using useEffect.

🟢 Mount
  • Component first renders
  • DOM is created
  • useEffect( fn, [] )
  • Fetch initial data
  • Set up subscriptions
🟡 Update
  • State or props changed
  • Component re-renders
  • useEffect( fn, [dep] )
  • Old cleanup runs first
  • New effect runs after
🔴 Unmount
  • Component removed
  • useEffect cleanup runs
  • Cancel timers
  • Unsubscribe listeners
  • Abort fetch requests
Lifecycle.jsx JSX 
import { useState, useEffect } from 'react'

function Timer() {
  const [seconds, setSeconds] = useState(0)

  useEffect(() => {
    console.log('🟢 Component MOUNTED — starting timer')

    const interval = setInterval(() => {
      setSeconds(prev => prev + 1)  // Functional update pattern
    }, 1000)

    // 🔴 Cleanup — runs on UNMOUNT
    return () => {
      console.log('🔴 Component UNMOUNTING — clearing timer')
      clearInterval(interval)
    }
  }, [])  // Empty array = run once on mount

  return <p>Time: {seconds}s</p>
}

Lists & Conditional Rendering

Render lists with .map(), and conditionally show content using &&, ternary operators, or early returns.

function ProductList() {

    </ul>
  )
}

Forms in React

React forms use controlled components — input values are driven by React state, making the component the "single source of truth".

LoginForm.jsx JSX 
import { useState } from 'react'

function LoginForm() {
  const [formData, setFormData] = useState({
    email: '',
    password: '',
    rememberMe: false
  })

  const [errors, setErrors] = useState({})

  // Generic handler — works for all inputs
  const handleChange = (e) => {
    const { name, value, type, checked } = e.target
    setFormData(prev => ({
      ...prev,
      [name]: type === 'checkbox' ? checked : value
    }))
  }

  const validate = () => {
    const newErrors = {}
    if (!formData.email) newErrors.email = 'Email is required'
    if (formData.password.length < 8) newErrors.password = 'Min 8 characters'
    return newErrors
  }

  const handleSubmit = (e) => {
    e.preventDefault()
    const validationErrors = validate()
    if (Object.keys(validationErrors).length > 0) {
      setErrors(validationErrors)
      return
    }
    console.log('Submitting:', formData)
  }

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label>Email</label>
        <input
          type="email"
          name="email"
          value={formData.email}      // controlled
          onChange={handleChange}
        />
        {errors.email && <p style={{color:'red'}}>{errors.email}</p>}
      </div>

      <div>
        <label>Password</label>
        <input
          type="password"
          name="password"
          value={formData.password}
          onChange={handleChange}
        />
      </div>

      <label>
        <input type="checkbox" name="rememberMe"
          checked={formData.rememberMe} onChange={handleChange} />
        Remember me
      </label>

      <button type="submit">Login</button>
    </form>
  )
}

Context API

Context solves "prop drilling" — passing props through many layers just to reach a deeply nested component. Context broadcasts data to any component that needs it.

🕳️ The Prop Drilling Problem

Without context: App → Layout → Sidebar → UserMenu → Avatar — each must pass user as a prop even if intermediate components don't use it.

With Context: any component can directly subscribe to the user data.

AuthContext.jsx JSX 
import { createContext, useContext, useState } from 'react'

// 1. Create the context
const AuthContext = createContext(null)

// 2. Create a Provider component
export function AuthProvider({ children }) {
  const [user, setUser] = useState(null)

  const login = (userData) => setUser(userData)
  const logout = () => setUser(null)

  return (
    {/* Wrap children — they all get access to this context */}
    <AuthContext.Provider value={{ user, login, logout }}>
      {children}
    </AuthContext.Provider>
  )
}

// 3. Custom hook for easy consumption
export function useAuth() {
  return useContext(AuthContext)
}

// 4. Wrap your app
// In main.jsx:
<AuthProvider>
  <App />
</AuthProvider>

// 5. Use anywhere in the tree!
function Avatar() {
  const { user, logout } = useAuth()   // No prop drilling!
  return (
    <div>
      <img src={user.avatar} alt={user.name} />
      <button onClick={logout}>Logout</button>
    </div>
  )
}

Patterns & Best Practices

Professional React code follows proven patterns. Learn these and your code will be cleaner, more maintainable, and easier to debug.

🪝 Custom Hooks

Extract reusable stateful logic into custom hooks. A custom hook is just a function that starts with use and calls other hooks.

hooks/useFetch.js JSX 
import { useState, useEffect } from 'react'

// Custom hook — reusable data fetching logic
export function useFetch(url) {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  useEffect(() => {
    const controller = new AbortController()

    fetch(url, { signal: controller.signal })
      .then(res => {
        if (!res.ok) throw new Error('Network error')
        return res.json()
      })
      .then(setData)
      .catch(err => {
        if (err.name !== 'AbortError') setError(err)
      })
      .finally(() => setLoading(false))

    return () => controller.abort()
  }, [url])

  return { data, loading, error }
}

// Usage — clean and reusable
function Posts() {
  const { data, loading, error } = useFetch('https://api.example.com/posts')

  if (loading) return <p>Loading...</p>
  if (error)   return <p>Error: {error.message}</p>
  return <ul>{data.map(p => <li key={p.id}>{p.title}</li>)}</ul>
}

✅ Do This

  • Keep components small & focused
  • Use descriptive variable names
  • Extract custom hooks for shared logic
  • Use stable IDs as list keys
  • Use functional state updates for derived state
  • Clean up effects (timers, subscriptions)
  • Co-locate state with the component that needs it
  • Use TypeScript for better type safety

❌ Avoid This

  • Mutating state directly
  • Using array index as key
  • Calling hooks conditionally
  • Missing dependency arrays in useEffect
  • Over-using global state
  • Deeply nested components
  • Forgetting event.preventDefault() on forms
  • Storing derived data in state

TodoApp.jsx — Complete Example JSX 
import { useState } from 'react'

// ── Child Component: Individual Todo Item ──
function TodoItem({ todo, onToggle, onDelete }) {
  return (
    <li style={{
      display: 'flex',
      gap: '10px',
      alignItems: 'center',
      textDecoration: todo.done ? 'line-through' : 'none',
      opacity: todo.done ? 0.5 : 1
    }}>
      <input
        type="checkbox"
        checked={todo.done}
        onChange={() => onToggle(todo.id)}
      />
      <span>{todo.text}</span>
      <span style={{fontSize:'0.75rem', color:'gray'}}>{todo.priority}</span>
      <button onClick={() => onDelete(todo.id)}>🗑️</button>
    </li>
  )
}

// ── Parent Component: App ──
function TodoApp() {
  const [todos, setTodos] = useState([
    { id: 1, text: 'Learn React', done: false, priority: '🔴 High' },
    { id: 2, text: 'Build a project', done: false, priority: '🟡 Mid' },
  ])
  const [text, setText] = useState('')
  const [filter, setFilter] = useState('all')  // all | active | done

  const addTodo = () => {
    if (!text.trim()) return
    setTodos([...todos, {
      id: Date.now(),
      text: text.trim(),
      done: false,
      priority: '🟢 Low'
    }])
    setText('')
  }

  const toggleTodo = (id) => {
    setTodos(todos.map(t => t.id === id ? { ...t, done: !t.done } : t))
  }

  const deleteTodo = (id) => {
    setTodos(todos.filter(t => t.id !== id))
  }

  // Derived data — don't store this in state!
  const filteredTodos = todos.filter(t => {
    if (filter === 'active') return !t.done
    if (filter === 'done') return t.done
    return true
  })

  const remaining = todos.filter(t => !t.done).length

  return (
    <div className="todo-app">
      <h1>⚛ React Todos</h1>
      <p>{remaining} tasks remaining</p>

      {/* Input */}
      <div>
        <input
          value={text}
          onChange={e => setText(e.target.value)}
          onKeyDown={e => e.key === 'Enter' && addTodo()}
          placeholder="Add a task..."
        />
        <button onClick={addTodo}>Add</button>
      </div>

      {/* Filter Buttons */}
      <div>
        {['all', 'active', 'done'].map(f => (
          <button key={f} onClick={() => setFilter(f)}
            style={{ fontWeight: filter === f ? 'bold' : 'normal' }}>
            {f}
          </button>
        ))}
      </div>

      {/* Todo List */}
      <ul>
        {filteredTodos.length === 0
          ? <p>No todos to show</p>
          : filteredTodos.map(todo => (
            <TodoItem
              key={todo.id}
              todo={todo}
              onToggle={toggleTodo}
              onDelete={deleteTodo}
            />
          ))
        }
      </ul>

      {/* Clear completed */}
      {todos.some(t => t.done) && (
        <button onClick={() => setTodos(todos.filter(t => !t.done))}>
          Clear Completed
        </button>
      )}
    </div>
  )
}

export default TodoApp
