Chapter 01

What is JavaScript?

Understanding the language of the web — where it lives, how it runs, and how to write your first line of code.

1.1 A Brief History

JavaScript was created by Brendan Eich in 1995 in just 10 days while he worked at Netscape. Originally called Mocha, then LiveScript, it was renamed JavaScript as a marketing move to piggyback on Java's popularity — though the two languages are entirely different.

Today, JavaScript is standardised under the name ECMAScript (ES). Modern browsers implement ES2015 (ES6) and beyond, giving developers a very powerful language to work with.

1.2 What Can JavaScript Do?

JavaScript is the only programming language that runs natively in the browser. It can:

  • Dynamically update HTML and CSS
  • Respond to user events (clicks, keypresses, scrolls)
  • Fetch data from servers without reloading a page (AJAX/Fetch)
  • Store data locally in the browser
  • Power full back-end servers via Node.js
  • Build mobile apps (React Native), desktop apps (Electron)

1.3 How JavaScript Runs

Browsers contain a JavaScript engine — a program that reads and executes your code. V8 (Chrome/Node.js), SpiderMonkey (Firefox), and JavaScriptCore (Safari) are the main ones. Modern engines use Just-In-Time (JIT) compilation to run JS at near-native speed.

JavaScript is single-threaded — it executes one task at a time, but its event loop allows it to handle asynchronous operations efficiently without blocking.

1.4 Adding JavaScript to a Web Page

There are three ways to include JavaScript in an HTML file:

Inline (avoid this)

<button onclick="alert('Hello!')">Click me</button>

Internal Script Tag

<script>
  // Your JS code goes here
  console.log('Hello, World!');
</script>

External File (recommended)

<!-- Place before </body> or use defer -->
<script src="app.js" defer></script>
Always use defer or place your script tag at the bottom of <body>. This ensures the HTML is fully parsed before JS runs, preventing errors when accessing DOM elements.

1.5 The Browser Console

Open DevTools (F12 or Cmd+Option+I) and go to the Console tab. This is your live playground for testing JavaScript.

// Print to the console
console.log('Hello, World!');     // general output
console.warn('Something is off');  // yellow warning
console.error('Something broke!'); // red error
console.table([1, 2, 3]);           // display as table
Chapter 02

Variables & Data Types

How JavaScript stores and categorises information — the building blocks of every program.

2.1 Declaring Variables

A variable is a named container for a value. JavaScript has three ways to declare one:

KeywordScopeRe-assignable?Hoisted?Use it?
varFunctionYesYes (as undefined)❌ Avoid
letBlockYesNo (TDZ)✅ Yes
constBlockNoNo (TDZ)✅ Preferred
 
const name = 'Tosin';   // value never changes
let age = 25;            // value may change later
age = 26;                // ✅ fine — let is re-assignable

// name = 'John';         ❌ TypeError: Assignment to constant variable
Default to const. Only switch to let when you know the value will be reassigned. Never use var in modern code.

2.2 Primitive Data Types

JavaScript has 7 primitive types — values that are immutable and stored directly.

// 1. String — text
const greeting = 'Hello';
const name = "World";
const msg = `Hello, ${name}!`; // template literal

// 2. Number — integers and decimals
const score = 98;
const pi = 3.14159;
const huge = 1e6;          // 1,000,000

// 3. Boolean
const isLoggedIn = true;
const hasPaid = false;

// 4. undefined — declared but not assigned
let city;
console.log(city); // undefined

// 5. null — intentional absence of a value
const user = null;

// 6. Symbol — unique identifier
const id = Symbol('id');

// 7. BigInt — integers beyond Number.MAX_SAFE_INTEGER
const bigNum = 9007199254740993n;

2.3 The typeof Operator

typeof 'hello'   // "string"
typeof 42        // "number"
typeof true      // "boolean"
typeof undefined // "undefined"
typeof null      // "object" ← famous JS bug, null is NOT an object
typeof {}        // "object"
typeof []        // "object" ← arrays are objects too
typeof function(){} // "function"

2.4 Type Coercion

JavaScript automatically converts types in some operations. This is called implicit coercion and is a common source of bugs.

// String + Number = String (concatenation wins)
'5' + 3        // "53"  ← NOT 8 !
'5' - 3        // 2    ← subtraction converts string to number

// Explicit conversion (always prefer this)
Number('42')   // 42
String(100)    // "100"
Boolean(0)     // false
Boolean('')    // false
Boolean('hi')  // true
parseInt('42px') // 42
Always use === (strict equality) instead of == (loose equality). Loose equality performs coercion: 0 == false is true, '' == false is true. Strict equality does not coerce.
Chapter 03

Operators

The symbols that perform computations, comparisons, and logical decisions in your code.

3.1 Arithmetic Operators

const a = 10, b = 3;

a + b   // 13  — addition
a - b   // 7   — subtraction
a * b   // 30  — multiplication
a / b   // 3.333... — division
a % b   // 1   — remainder (modulo)
a ** b  // 1000 — exponentiation (10³)

// Increment / Decrement
let count = 5;
count++;  // 6 (post-increment)
++count;  // 7 (pre-increment)
count--;  // 6 (post-decrement)

3.2 Assignment Operators

let x = 10;
x += 5;   // x = x + 5  → 15
x -= 3;   // x = x - 3  → 12
x *= 2;   // x = x * 2  → 24
x /= 4;   // x = x / 4  → 6
x %= 4;   // x = x % 4  → 2
x **= 3;  // x = x ** 3 → 8

3.3 Comparison Operators

5 === 5    // true  — strict equal (type + value)
5 !== 6    // true  — strict not equal
5 > 3      // true
5 >= 5     // true
5 < 10     // true
5 <= 4     // false

3.4 Logical Operators

// AND — both must be true
true && true    // true
true && false   // false

// OR — at least one must be true
false || true   // true
false || false  // false

// NOT — inverts the boolean
!true           // false
!false          // true

// Practical use
const isAdmin = true;
const hasAccess = true;
if (isAdmin && hasAccess) console.log('Welcome!');

3.5 Nullish Coalescing & Optional Chaining

// ?? — use right side only if left is null or undefined
const username = null ?? 'Guest';   // "Guest"
const port = 0 ?? 3000;             // 0  (0 is not null/undefined)

// ?. — optional chaining: access deeply without throwing
const user = { profile: { name: 'Tosin' } };
const city = user?.address?.city;    // undefined (no error!)
const name = user?.profile?.name;    // "Tosin"

3.6 Ternary Operator

A compact one-line if/else:

const age = 20;
const status = age >= 18 ? 'Adult' : 'Minor';
// condition ? value_if_true : value_if_false
console.log(status); // "Adult"
Chapter 04

Control Flow

Making your program make decisions and repeat actions based on conditions.

4.1 if / else if / else

const score = 72;

if (score >= 90) {
  console.log('A');
} else if (score >= 80) {
  console.log('B');
} else if (score >= 70) {
  console.log('C'); // ← this runs
} else {
  console.log('F');
}

4.2 switch Statement

const day = 'Monday';

switch (day) {
  case 'Monday':
  case 'Tuesday':
    console.log('Early week');
    break;
  case 'Friday':
    console.log('TGIF!');
    break;
  default:
    console.log('Mid week');
}
Always include break in switch cases. Without it, execution "falls through" into the next case — this is usually a bug.

4.3 Loops

for Loop

// for (initializer; condition; update)
for (let i = 0; i < 5; i++) {
  console.log(`Iteration ${i}`);
}
// Prints: Iteration 0, 1, 2, 3, 4

while Loop

let count = 0;
while (count < 3) {
  console.log(`count is ${count}`);
  count++;
}

for...of (arrays, strings)

const fruits = ['apple', 'mango', 'grape'];

for (const fruit of fruits) {
  console.log(fruit); // apple, mango, grape
}

for...in (objects)

const person = { name: 'Tosin', age: 25, city: 'Lagos' };

for (const key in person) {
  console.log(`${key}: ${person[key]}`);
}
// name: Tosin  /  age: 25  /  city: Lagos

4.4 break & continue

for (let i = 0; i < 10; i++) {
  if (i === 3) continue; // skip 3
  if (i === 7) break;    // stop at 7
  console.log(i);          // 0, 1, 2, 4, 5, 6
}
Chapter 05

Functions

Reusable blocks of code — the most fundamental building blocks of any JavaScript program.

5.1 Function Declaration

function greet(name) {
  return `Hello, ${name}!`;
}

console.log(greet('Tosin')); // "Hello, Tosin!"

// Function declarations are hoisted — you can call them before they're defined

5.2 Function Expression

const greet = function(name) {
  return `Hello, ${name}!`;
};

// NOT hoisted — must be defined before calling

5.3 Arrow Functions (ES6)

A concise syntax for writing functions, especially useful for callbacks:

// Full form
const add = (a, b) => {
  return a + b;
};

// Implicit return (single expression, no braces)
const multiply = (a, b) => a * b;

// Single parameter (no parentheses needed)
const double = n => n * 2;

// No parameters
const sayHi = () => 'Hi!';

console.log(add(3, 4));      // 7
console.log(double(6));     // 12

5.4 Parameters & Arguments

// Default parameters
function createUser(name, role = 'viewer') {
  return { name, role };
}
createUser('Ada');           // { name: 'Ada', role: 'viewer' }
createUser('Tosin', 'admin'); // { name: 'Tosin', role: 'admin' }

// Rest parameters — collects remaining args into an array
function sum(...numbers) {
  return numbers.reduce((acc, n) => acc + n, 0);
}
sum(1, 2, 3, 4); // 10

5.5 First-Class Functions & Callbacks

In JavaScript, functions are first-class citizens — they can be stored in variables, passed as arguments, and returned from other functions.

// Passing a function as an argument (callback)
function runTwice(fn) {
  fn();
  fn();
}

runTwice(() => console.log('Hello!'));
// Hello!  Hello!

// Returning a function from a function
function multiplier(factor) {
  return n => n * factor;
}
const triple = multiplier(3);
triple(5); // 15
A function that takes or returns another function is called a Higher-Order Function. Array methods like .map(), .filter(), and .reduce() are all higher-order functions.

Chapter 06

Arrays

Ordered collections of data — and the powerful methods for transforming them.

6.1 Creating & Accessing Arrays

const colors = ['red', 'green', 'blue'];

colors[0]     // 'red'     — zero-indexed
colors[2]     // 'blue'
colors.length // 3
colors.at(-1) // 'blue'   — last element (ES2022)

// Modifying
colors.push('yellow');   // add to end   → ['red','green','blue','yellow']
colors.pop();            // remove last  → ['red','green','blue']
colors.unshift('pink'); // add to front
colors.shift();          // remove first

6.2 Essential Array Methods

forEach — iterate

[1, 2, 3].forEach(n => console.log(n * 2)); // 2, 4, 6

map — transform each element

const prices = [10, 20, 30];
const withTax = prices.map(p => p * 1.1);
// [11, 22, 33]  — returns a new array

filter — keep matching elements

const nums = [1, 2, 3, 4, 5, 6];
const evens = nums.filter(n => n % 2 === 0);
// [2, 4, 6]

reduce — accumulate into a single value

const total = [10, 20, 30].reduce((acc, val) => acc + val, 0);
// 60

find & findIndex

const users = [
  { id: 1, name: 'Ada' },
  { id: 2, name: 'Tosin' }
];
const user = users.find(u => u.id === 2);
// { id: 2, name: 'Tosin' }

const idx = users.findIndex(u => u.id === 2);
// 1

some & every

const scores = [55, 72, 88, 41];
scores.some(s => s > 80);   // true  — at least one
scores.every(s => s > 40);  // true  — all of them

flat & flatMap

[1, [2, [3]]].flat(2)                   // [1, 2, 3]
[1, 2, 3].flatMap(n => [n, n * 2])      // [1, 2, 2, 4, 3, 6]

6.3 Spread & Destructuring

// Spread — copy / merge arrays
const a = [1, 2];
const b = [3, 4];
const combined = [...a, ...b]; // [1, 2, 3, 4]

// Destructuring — extract values
const [first, second, ...rest] = [10, 20, 30, 40];
// first=10, second=20, rest=[30, 40]
Chapter 07

Objects

Key-value stores that model real-world entities — the heart of JavaScript data structures.

7.1 Creating Objects

// Object literal (most common)
const person = {
  name: 'Tosin',
  age: 25,
  city: 'Lagos',
  greet() {
    return `Hi, I'm ${this.name}`;
  }
};

// Accessing properties
person.name        // 'Tosin'   — dot notation
person['age']      // 25         — bracket notation
person.greet()     // "Hi, I'm Tosin"

// Adding / updating
person.email = 'k@example.com';
person.age = 26;

// Deleting
delete person.city;

7.2 Object Destructuring

const { name, age, city = 'Unknown' } = person;
// Extracts properties into variables; default for missing ones

// Rename while destructuring
const { name: fullName } = person;
console.log(fullName); // 'Tosin'

// In function parameters
function display({ name, age }) {
  console.log(`${name} is ${age}`);
}
display(person);

7.3 Spread & Object.assign

// Spread to merge objects
const defaults = { theme: 'light', lang: 'en' };
const settings = { ...defaults, theme: 'dark' };
// { theme: 'dark', lang: 'en' }  — later key wins

// Immutable update pattern
const updatedPerson = { ...person, age: 27 };

7.4 Useful Object Methods

const car = { make: 'Toyota', model: 'Camry', year: 2022 };

Object.keys(car)    // ['make', 'model', 'year']
Object.values(car)  // ['Toyota', 'Camry', 2022]
Object.entries(car) // [['make','Toyota'], ['model','Camry'], ...]

// Iterate over entries
Object.entries(car).forEach(([key, val]) => {
  console.log(`${key}: ${val}`);
});

// Freeze an object (prevent changes)
const config = Object.freeze({ apiUrl: '/api' });
Chapter 08

The DOM

The Document Object Model — JavaScript's interface to everything visible on a web page.

8.1 What is the DOM?

When a browser loads an HTML page, it builds a tree-like structure in memory called the DOM (Document Object Model). JavaScript uses this tree to read, create, update, and delete HTML elements and CSS styles dynamically.

8.2 Selecting Elements

// Single element selectors
document.getElementById('header')       // by ID
document.querySelector('.card')        // first match (CSS selector)
document.querySelector('#nav > a')     // nested selector

// Multiple elements
document.querySelectorAll('li')         // NodeList of all <li>
document.getElementsByClassName('btn') // HTMLCollection

// querySelectorAll returns a NodeList — convert to Array if needed
const items = [...document.querySelectorAll('li')];
items.forEach(li => console.log(li.textContent));

8.3 Reading & Changing Content

const el = document.querySelector('#title');

// Reading
el.textContent  // plain text (safe)
el.innerHTML    // HTML string (⚠️ XSS risk with user input)

// Writing
el.textContent = 'New Title';
el.innerHTML = '<strong>Bold Title</strong>';

8.4 Changing Styles & Classes

const box = document.querySelector('.box');

// Inline styles
box.style.backgroundColor = 'coral';
box.style.fontSize = '18px';

// Class manipulation (preferred — keeps CSS in stylesheet)
box.classList.add('active');
box.classList.remove('hidden');
box.classList.toggle('dark');    // add if absent, remove if present
box.classList.contains('active'); // true / false

8.5 Creating & Inserting Elements

// Create a new element
const li = document.createElement('li');
li.textContent = 'New Item';
li.classList.add('item');

// Insert into the DOM
const list = document.querySelector('#list');
list.appendChild(li);            // at end
list.prepend(li);               // at beginning
list.insertAdjacentHTML('beforeend', '<li>Another</li>');

// Remove an element
li.remove();

8.6 Attributes

const img = document.querySelector('img');

img.getAttribute('src')           // get attribute
img.setAttribute('alt', 'Logo')   // set attribute
img.removeAttribute('alt')       // remove attribute

// data-* attributes
// <button data-id="42">Delete</button>
const btn = document.querySelector('button');
btn.dataset.id // "42"
Chapter 09

Events

Listening and responding to user interactions — the foundation of interactivity.

9.1 addEventListener

const btn = document.querySelector('#myBtn');

btn.addEventListener('click', function(event) {
  console.log('Clicked!', event);
});

// Arrow function shorthand
btn.addEventListener('click', (e) => {
  e.target.textContent = 'Clicked!';
});

9.2 The Event Object

document.addEventListener('click', (e) => {
  console.log(e.target);        // element clicked
  console.log(e.type);          // 'click'
  console.log(e.clientX, e.clientY); // mouse coords
});

document.addEventListener('keydown', (e) => {
  console.log(e.key);    // 'Enter', 'a', 'ArrowUp', etc.
  console.log(e.code);   // 'KeyA', 'Enter'
  if (e.key === 'Enter') console.log('Enter pressed!');
});

9.3 Preventing Default Behaviour

// Stop a form from submitting (page reload)
form.addEventListener('submit', (e) => {
  e.preventDefault();
  // now handle the data yourself
});

// Stop a link from navigating
link.addEventListener('click', (e) => {
  e.preventDefault();
  console.log('Navigation blocked');
});

9.4 Event Delegation

Instead of attaching an event to every child element, attach one listener to the parent. This is more efficient and works for dynamically added elements.

// Instead of this (bad for large lists):
// document.querySelectorAll('li').forEach(li => li.addEventListener(...))

// Do this (one listener on the parent):
document.querySelector('#list').addEventListener('click', (e) => {
  if (e.target.matches('li')) {
    console.log('Clicked item:', e.target.textContent);
  }
});

Chapter 10

Forms & Validation

Reading form data, responding to user input, and enforcing data integrity.

10.1 Reading Form Values

<!-- HTML -->
<form id="signupForm">
  <input type="text" id="username" placeholder="Username" />
  <input type="email" id="email" placeholder="Email" />
  <button type="submit">Sign Up</button>
</form>
// JavaScript
const form = document.getElementById('signupForm');

form.addEventListener('submit', (e) => {
  e.preventDefault();

  const username = document.getElementById('username').value.trim();
  const email    = document.getElementById('email').value.trim();

  // You can also use FormData
  const data = new FormData(form);
  console.log(data.get('username'));
});

10.2 Validation

function validate(username, email) {
  const errors = [];

  if (!username || username.length < 3) {
    errors.push('Username must be at least 3 characters');
  }

  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!emailRegex.test(email)) {
    errors.push('Please enter a valid email');
  }

  return errors;
}

const errs = validate('Jo', 'bad-email');
if (errs.length > 0) {
  errs.forEach(e => console.log(e));
}
HTML5 has built-in validation attributes like required, minlength, pattern, and type="email". Use these as your first line of defence, then add JS validation for better UX and custom messages.

ES6+ Features

The modern JavaScript syntax that makes code cleaner, safer, and more expressive.

11.1 Template Literals

const name = 'Tosin';
const role = 'developer';

// Multi-line strings with expressions
const html = `
  <div class="card">
    <h2>${name}</h2>
    <p>${role.toUpperCase()}</p>
    <p>Score: ${95 * 1.1}</p>
  </div>
`;

11.2 Destructuring (Recap + Advanced)

// Nested destructuring
const user = {
  name: 'Ada',
  address: { city: 'Abuja', zip: '900001' }
};
const { name, address: { city } } = user;
console.log(city); // 'Abuja'

// Swap variables
let a = 1, b = 2;
[a, b] = [b, a];
// a=2, b=1

11.3 Spread & Rest Operators

// Spread in function calls
const nums = [3, 1, 4, 1, 5];
Math.max(...nums); // 5

// Rest in function params
function log(level, ...messages) {
  messages.forEach(m => console.log(`[${level}] ${m}`));
}
log('INFO', 'Server started', 'Listening on 3000');

11.4 Short-circuit Evaluation

// && — return right side if left is truthy
const user = { name: 'Tosin' };
const label = user && user.name; // 'Tosin'

// || — return right side if left is falsy
const port = process?.env?.PORT || 3000;

// Practical: conditional render in template literals
const isAdmin = true;
const html = `${isAdmin && '<button>Delete</button>'}`;

11.5 Map & Set

// Map — key-value pairs where keys can be any type
const map = new Map();
map.set('user', { name: 'Tosin' });
map.set(42, 'the answer');
map.get('user'); // { name: 'Tosin' }
map.size;       // 2

// Set — unique values only
const set = new Set([1, 2, 2, 3, 3]);
[...set] // [1, 2, 3]  — duplicates removed!

// Remove duplicates from an array
const unique = [...new Set([1,1,2,3,3])]; // [1, 2, 3]
Chapter 12

Scope & Closures

Understanding where variables live and one of JavaScript's most powerful — and misunderstood — features.

12.1 Scope

Scope determines where a variable can be accessed in your code.

// Global scope — accessible everywhere
const SITE_NAME = 'MyApp';

function outer() {
  // Function scope
  const outerVar = 'I am outer';

  if (true) {
    // Block scope (let/const only)
    const blockVar = 'I am block-scoped';
    console.log(outerVar);  // ✅ accessible
  }

  // console.log(blockVar); ❌ ReferenceError
}

// console.log(outerVar); ❌ ReferenceError

12.2 Closures

A closure is a function that remembers the variables from its outer scope even after the outer function has finished executing.

function makeCounter() {
  let count = 0; // this is "closed over"

  return {
    increment() { count++; },
    decrement() { count--; },
    getCount()  { return count; }
  };
}

const counter = makeCounter();
counter.increment();
counter.increment();
counter.increment();
counter.decrement();
console.log(counter.getCount()); // 2

// 'count' is private! It cannot be accessed directly.
Closures are used everywhere in JavaScript: for data privacy, in callbacks, event handlers, currying, and module patterns. Mastering closures unlocks a deeper level of JavaScript fluency.

12.3 Practical Closure Use Cases

// 1. Memoisation — cache expensive results
function memoize(fn) {
  const cache = {};
  return (...args) => {
    const key = JSON.stringify(args);
    if (cache[key]) return cache[key];
    cache[key] = fn(...args);
    return cache[key];
  };
}

// 2. Once — a function that runs only once
function once(fn) {
  let called = false;
  return (...args) => {
    if (!called) { called = true; return fn(...args); }
  };
}
const init = once(() => console.log('Initialised!'));
init(); // "Initialised!"
init(); // nothing
Chapter 13

Prototypes & Classes

Object-oriented programming in JavaScript — blueprints for creating structured, reusable objects.

13.1 ES6 Classes

class Animal {
  // constructor runs when you use `new`
  constructor(name, sound) {
    this.name = name;
    this.sound = sound;
  }

  // instance method
  speak() {
    return `${this.name} says ${this.sound}!`;
  }

  // static method — called on the class, not an instance
  static kingdom() {
    return 'Animalia';
  }
}

const dog = new Animal('Dog', 'Woof');
dog.speak();         // "Dog says Woof!"
Animal.kingdom();   // "Animalia"

13.2 Inheritance with extends

class Dog extends Animal {
  constructor(name, breed) {
    super(name, 'Woof'); // call parent constructor
    this.breed = breed;
  }

  fetch(item) {
    return `${this.name} fetches the ${item}!`;
  }

  // Override parent method
  speak() {
    return `${super.speak()} (a ${this.breed})`;
  }
}

const rex = new Dog('Rex', 'Labrador');
rex.speak();   // "Rex says Woof! (a Labrador)"
rex.fetch('ball'); // "Rex fetches the ball!"
rex instanceof Dog;    // true
rex instanceof Animal; // true

class BankAccount {
  #balance = 0; // private — only accessible inside the class

  deposit(amount) {
    if (amount > 0) this.#balance += amount;
  }

  getBalance() {
    return this.#balance;
  }
}

const account = new BankAccount();
account.deposit(100);

Error Handling

Anticipating, catching, and recovering from errors gracefully.

14.1 try / catch / finally

try {
  // Code that might throw an error
  const data = JSON.parse('invalid json {'); // throws!
  console.log(data);
} catch (error) {
  // Runs if an error was thrown
  console.error('Caught:', error.message);
} finally {
  // Always runs — great for cleanup
  console.log('Done (with or without error)');
}

14.2 Throwing Custom Errors

function divide(a, b) {
  if (b === 0) {
    throw new Error('Division by zero is not allowed');
  }
  return a / b;
}

try {
  console.log(divide(10, 0));
} catch (e) {
  console.error(e.message); // "Division by zero is not allowed"
}

14.3 Custom Error Classes

class ValidationError extends Error {
  constructor(message, field) {
    super(message);
    this.name = 'ValidationError';
    this.field = field;
  }
}

function validateAge(age) {
  if (age < 0 || age > 120) {
    throw new ValidationError('Age must be 0–120', 'age');
  }
}

try {
  validateAge(-5);
} catch (e) {
  if (e instanceof ValidationError) {
    console.log(`Field "${e.field}": ${e.message}`);
  }
}

Chapter 15

Callbacks & Promises

Handling operations that take time — from the old callback pattern to the clean Promise API.

15.1 The Problem: Async Operations

JavaScript is single-threaded. Operations like network requests, file reads, or timers take time. If they blocked execution, the whole page would freeze. JavaScript uses an event loop to handle these asynchronously.

console.log('1');

setTimeout(() => {
  console.log('2'); // runs after current stack clears
}, 0);

console.log('3');

// Output: 1, 3, 2

15.2 Callbacks (the old way)

function fetchData(id, callback) {
  setTimeout(() => {
    const data = { id, name: 'User' };
    callback(null, data); // Node.js convention: (error, result)
  }, 1000);
}

fetchData(1, (err, data) => {
  if (err) return console.error(err);
  console.log(data);
});

// Problem: Callback Hell (Pyramid of Doom)
fetchUser(id, (err, user) => {
  fetchPosts(user.id, (err, posts) => {
    fetchComments(posts[0].id, (err, comments) => {
      // deeply nested, hard to read
    });
  });
});

15.3 Promises

A Promise represents a value that will be available in the future. It can be pending, fulfilled, or rejected.

// Creating a promise
function delay(ms) {
  return new Promise((resolve, reject) => {
    setTimeout(resolve, ms);
  });
}

// Consuming a promise
delay(1000)
  .then(() => console.log('1 second later!'))
  .catch(err => console.error(err))
  .finally(() => console.log('Done'));

15.4 Promise Chaining

fetchUser(1)
  .then(user => fetchPosts(user.id))
  .then(posts => fetchComments(posts[0].id))
  .then(comments => console.log(comments))
  .catch(err => console.error('Something failed:', err));

15.5 Promise.all, Promise.race, Promise.allSettled

// Run multiple promises in parallel — wait for ALL
const [users, posts] = await Promise.all([
  fetch('/api/users').then(r => r.json()),
  fetch('/api/posts').then(r => r.json()),
]);

// Finish as soon as the FIRST settles
const fastest = await Promise.race([p1, p2, p3]);

// Wait for ALL — don't throw if one fails
const results = await Promise.allSettled([p1, p2, p3]);
results.forEach(r => {
  if (r.status === 'fulfilled') console.log(r.value);
  else console.error(r.reason);
});
Chapter 16

Async / Await

Syntactic sugar over Promises — write asynchronous code that reads like synchronous code.

16.1 The async / await Syntax

// Mark the function with `async`
async function getUser(id) {
  // `await` pauses here until the promise resolves
  const user = await fetchUser(id);
  const posts = await fetchPosts(user.id);

  return { user, posts };
}

// Arrow function form
const getData = async () => {
  const data = await someAsyncOp();
  return data;
};
An async function always returns a Promise, even if you return a plain value. await can only be used inside an async function (or at the top level of a module in modern environments).

16.2 Error Handling with async/await

async function loadData() {
  try {
    const response = await fetch('/api/data');

    if (!response.ok) {
      throw new Error(`HTTP error: ${response.status}`);
    }

    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Failed to load:', error.message);
    throw error; // re-throw if caller needs to handle it
  }
}

16.3 Parallel vs Sequential

// ❌ Sequential — slow, waits for each one
const user  = await fetchUser();   // 1s
const posts = await fetchPosts();  // 1s — total: 2s

// ✅ Parallel — fast, both run at the same time
const [user, posts] = await Promise.all([
  fetchUser(),
  fetchPosts()
]); // total: ~1s

17.3 A Reusable HTTP Utility

const http = {
  async get(url) {
    const res = await fetch(url);
    if (!res.ok) throw new Error(res.statusText);
    return res.json();
  },

  async post(url, data) {
    const res = await fetch(url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    });
    if (!res.ok) throw new Error(res.statusText);
    return res.json();
  }
};

// Usage

const post  = await http.post('/api/posts', { title: 'Hello' });

  }

Same API as localStorage, but data is cleared when the browser tab is closed.

19.3 A Type-Safe Storage Utility

const storage = {
  get(key, fallback = null) {
    try {
      const val = localStorage.getItem(key);
      return val ? JSON.parse(val) : fallback;
    } catch { return fallback; }
  },
  set(key, value) {
    localStorage.setItem(key, JSON.stringify(value));
  },
  remove(key) {
    localStorage.removeItem(key);
  }
};

// Usage

Performance & Best Practices

Writing code that is fast, readable, and maintainable at scale.

20.1 Debounce & Throttle

Limit how often expensive functions run in response to frequent events.

// Debounce — only runs AFTER the user stops triggering for `delay` ms
// Use case: search-as-you-type, resize handlers
function debounce(fn, delay) {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

const search = debounce((query) => fetchResults(query), 400);
input.addEventListener('input', e => search(e.target.value));

// Throttle — runs at most once every `limit` ms
// Use case: scroll events, infinite scroll, analytics
function throttle(fn, limit) {
  let inThrottle;
  return (...args) => {
    if (!inThrottle) {
      fn(...args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}

20.2 DOM Performance

// ❌ Slow — multiple reflows
for (let i = 0; i < 100; i++) {
  document.body.appendChild(document.createElement('div'));
}

// ✅ Fast — single reflow with DocumentFragment
const fragment = document.createDocumentFragment();
for (let i = 0; i < 100; i++) {
  fragment.appendChild(document.createElement('div'));
}
document.body.appendChild(fragment);
